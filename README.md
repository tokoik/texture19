# texture19 - 第２３回 放物面マッピングと他のテクスチャとの合成 サンプルプログラム

## 1. 概要

このプログラムは、OpenGL における「マルチテクスチャリング (Multitexturing)」の基礎を学ぶための、学生向けのサンプルプログラムです。本プログラムは、以下のブログ記事の解説に沿って学習を進めるための雛形として提供されています。

- [第２２回 放物面マッピング](https://tokoik.github.io/blog/%E3%83%86%E3%82%AF%E3%82%B9%E3%83%81%E3%83%A3%E5%85%A5%E9%96%80/2005/06/24/texture.html)

このプログラムは[放物面マッピングのサンプルプログラム](https://github.com/tokoik/texture18)をもとに、放物面マッピングにさらに別のテクスチャを合成する方法について解説するものです。

## 2. ビルド方法

このプログラムは CMake を用いてビルドを構成しています。各プラットフォームごとの手順は以下の通りです。なお、プログラムをビルドするためのバイナリディレクトリは、バージョン管理ファイル（.gitignore）の設定に合わせて build という名前にします。

### 2.1 Windows (Visual Studio 2022 の場合)

1. コマンドプロンプトまたは PowerShell を開き、このプロジェクトのディレクトリに移動します。
2. 以下のコマンドを実行してビルドディレクトリを作成し、CMake で構成を行います。

   ```bat
   mkdir build
   cd build
   cmake .. -G "Visual Studio 17 2022"
   ```

3. 生成された build フォルダ内の texture19.sln を Visual Studio で開きます。
4. ソリューションエクスプローラーで texture19 プロジェクトを右クリックし、「スタートアップ プロジェクトに設定」を選択します。
5. 「ローカル Windows デバッガー」をクリックするか、F5 キーを押してビルドおよび実行します。

### 2.2 macOS (Xcode の場合)

1. ターミナルを開き、このプロジェクトのディレクトリに移動します。
2. 以下のコマンドを実行してビルドディレクトリを作成し、Xcode 用のプロジェクトを生成します。

   ```sh
   mkdir build
   cd build
   cmake .. -G Xcode
   ```

3. 生成された build/texture19.xcodeproj を Xcode で開きます。
4. 左上のスキーム選択（再生ボタンの横）が texture19 になっていることを確認します。
5. 「Run」ボタン（再生ボタン）をクリックするか、Command + R を押してビルドおよび実行します。

### 2.3 Ubuntu Linux

1. ターミナルを開き、このプロジェクトのディレクトリに移動します。
2. 必要なパッケージ（freeglut3-dev や pkg-config など）がインストールされていることを確認し、以下のコマンドでビルドします。

   ```sh
   mkdir build
   cd build
   cmake ..
   make
   ```

## 3. 使い方

### 3.1 プログラムの起動方法

各OSとも、ビルド後に生成されるバイナリディレクトリ (build) やそのサブフォルダから起動します。（※ CMake の設定により、Windows や Xcode では Debug などのフォルダ下に実行ファイルが置かれることがあります）

- **Windows**

  Visual Studio 上で「ローカル Windows デバッガー」をクリックして実行するか、またはコマンドプロンプトから以下のコマンドで起動します。

  ```cmd
  cd build\Debug
  texture19.exe
  ```

- **macOS**

  Xcode 上で左上の「Run（再生ボタン）」をクリックするのが楽です。これにより texture19.app アプリケーションバンドルとして自動的に実行されます。アプリケーションバンドルを直接起動するなら、Finder から build/Debug/texture19.app をダブルクリックするか、ターミナルから open build/Debug/texture19.app を実行します (この場合はエラーメッセージ等が表示されません)。

- **Ubuntu Linux**

  ターミナルから以下のコマンドで実行ファイル（バイナリ）を直接起動します。

  ```sh
  cd build
  ./texture19
  ```

### 3.2 操作方法

- **マウスの左ボタンでドラッグ**

  物体（デフォルトでは箱）を回転させることができます。放物面マッピングによる周囲の環境の映り込みと、下地のテクスチャの合成結果を確認できます。

- **キーボードの q, Q または ESC キー**

  プログラムを終了します。

## 4. 解説

プログラムのソースコード（主に `main.cpp` と `box.cpp`）をもとに、このプログラム内で行われている処理手順を解説します。このプログラムでは、放物面マッピングによる全方位の映り込み（環境マッピング）に加えて、さらに下地のテクスチャを合成する処理を実装しています。

### 4.1 テクスチャユニットの割り当ての変更

OpenGL のマルチテクスチャはユニット番号の順に合成処理が行われます。下地のテクスチャの上に不透明な環境マッピングのテクスチャを重ねると、下地が完全に隠れてしまいます。

これを解決するため、本プログラムではテクスチャユニットの割り当てを以下のように変更し、環境マッピングのテクスチャを先に適用した上で、最後に下地のテクスチャを合成するようにしています。

- **裏面の放物面テクスチャ**

  `GL_TEXTURE1` → `GL_TEXTURE0`

- **表面の放物面テクスチャ**

  `GL_TEXTURE2` → `GL_TEXTURE1`

- **下地のテクスチャ**

  `GL_TEXTURE0` → `GL_TEXTURE2`

### 4.2 テクスチャの読み込みと設定 (`init()` 関数)

1. **裏面の放物面テクスチャ (テクスチャユニット0)**

   `glActiveTexture(GL_TEXTURE0)` を呼び出し、裏面用の画像 `paraboloid1.raw` を割り当てます。テクスチャ環境は `GL_REPLACE` に設定して下地を置き換え、反射ベクトルをテクスチャ座標として自動生成します。

2. **表面の放物面テクスチャ (テクスチャユニット1)**

   `glActiveTexture(GL_TEXTURE1)` に切り替え、表面用の画像 `paraboloid2.raw` を割り当てます。テクスチャ環境は `GL_DECAL` に指定し、画像のアルファ値を用いて裏面のテクスチャ上に必要な部分だけを上書き合成します。

3. **下地のテクスチャ (テクスチャユニット2)**

   `glActiveTexture(GL_TEXTURE2)` に切り替え、`dot.raw` の画像を割り当てます。テクスチャ環境モードを **`GL_COMBINE`** とし、合成方法に **`GL_INTERPOLATE`** を指定しています。これにより、`GL_TEXTURE_ENV_COLOR` で指定したアルファ値（このプログラムでは 0.5）にもとづいて、テクスチャユニット1までの合成結果（放物面マッピング）と下地のテクスチャが線形補間で半透明に合成されます。

### 4.3 特定のテクスチャユニットに対するテクスチャ座標の指定

下地のテクスチャは自動生成ではなく、図形の各頂点に対して明示的にテクスチャ座標を与える必要があります（`box.cpp` を参照）。しかし、標準の `glTexCoord2dv()` 関数はデフォルトのユニット（ユニット0）に対してしか作用しません。

そこで、特定のテクスチャユニット（ここではテクスチャユニット２）に対してテクスチャ座標を設定するために、**`glMultiTexCoord2dv()`** という関数を使用します。

```cpp
/* テクスチャ座標の指定 */
glMultiTexCoord2dv(GL_TEXTURE2, texcoord[j][i]);
```

> **補足 (Windows の場合):**
> `glMultiTexCoord*()` は OpenGL 1.3 以降の機能であるため、Windows 標準の `gl.h` には宣言が含まれていません。そのため、`glext.h` をインクルードし、`wglGetProcAddress()` を用いて関数のポインタを取得してから使用しています。

### 4.4 描画時のマルチテクスチャの有効化 (`scene()` 関数)

図形を描画する際には、各テクスチャユニットを順次アクティブにして機能を有効化（`glEnable()`）します。ユニット0とユニット1では `GL_TEXTURE_2D` に加えてテクスチャ座標の自動生成（`GL_TEXTURE_GEN_S` など）を有効にし、ユニット2では `GL_TEXTURE_2D` のみを有効にします。描画処理（`box()`）が完了したら、他の描画に影響を与えないよう `glDisable()` を使って各機能を無効化します。
